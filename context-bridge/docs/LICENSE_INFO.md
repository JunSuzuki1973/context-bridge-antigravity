## English

### Why MIT License?

This project uses the **MIT License**, one of the most permissive and widely-used open source licenses.

**Reasons for choosing MIT**:

✅ **Simple and Clear**: Easy to understand, only a few lines  
✅ **Maximum Freedom**: Commercial use, modification, distribution all allowed  
✅ **No Copyleft**: Derivative works don't need to be open source  
✅ **Industry Standard**: Most popular license on GitHub/npm  
✅ **Business Friendly**: Easily integrated into commercial products  

**What you CAN do**:
- ✅ Use commercially
- ✅ Modify the code
- ✅ Distribute
- ✅ Use privately
- ✅ Create closed-source derivatives

**What you MUST do**:
- ⚠️ Include the original license and copyright notice

**What we are NOT liable for**:
- ❌ No warranty provided
- ❌ Not liable for damages

### Alternative Licenses Considered

| License | Pros | Cons | Verdict |
|---------|------|------|---------|
| **MIT** ✅ | Simple, permissive, popular | No patent protection | **Selected** |
| Apache 2.0 | Patent grant, contributor protection | More complex | Good alternative |
| GPL v3 | Strong copyleft | Restricts commercial use | Too restrictive |
| BSD 3-Clause | Similar to MIT | Less popular | Not needed |

### If You Want to Change the License

If you prefer **Apache 2.0** for patent protection:

1. Replace `LICENSE` file with Apache 2.0 text
2. Update README badges
3. Add NOTICE file if required

## 日本語

### なぜMITライセンス？

このプロジェクトは、最も寛容で広く使用されているオープンソースライセンスの一つ、**MITライセンス**を採用しています。

**MITを選んだ理由**:

✅ **シンプルで明確**: 理解しやすく、数行のみ  
✅ **最大の自由**: 商用利用、改変、配布すべて許可  
✅ **コピーレフトなし**: 派生物をオープンソースにする必要なし  
✅ **業界標準**: GitHub/npmで最も人気  
✅ **ビジネスフレンドリー**: 商用製品に簡単に組み込める  

**できること**:
- ✅ 商用利用
- ✅ コードの改変
- ✅ 配布
- ✅ 私的利用
- ✅ クローズドソースの派生物作成

**必ずすべきこと**:
- ⚠️ 元のライセンスと著作権表示を含める

**責任の範囲**:
- ❌ 保証は提供されません
- ❌ 損害に対する責任なし

### 検討した代替ライセンス

| ライセンス | 長所 | 短所 | 判定 |
|---------|------|------|------|
| **MIT** ✅ | シンプル、寛容、人気 | 特許保護なし | **採用** |
| Apache 2.0 | 特許許諾、貢献者保護 | やや複雑 | 良い代替案 |

1. `LICENSE`ファイルをApache 2.0のテキストに置き換え
2. READMEのバッジを更新
3. 必要に応じてNOTICEファイルを追加
